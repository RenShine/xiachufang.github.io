@all

如果对某些页面有统计需求（比如 simplecms 页面、supercms 页面），投放广告、发送 push 或者设置分享链接的时候，可以在 URL 后面加 utm_campaign（广告名称） utm_medium（广告媒介） utm_source （广告来源）几个参数，之后可以在 Google Analytics 和 GrowingIO 里通过“广告来源”“广告媒介”“广告名称”等若干 维度 分析用户群体和页面访问。

⚠️ 如果 URL 是在 app 内打开而且是 app 原生支持的页面（菜谱、菜单、作品、话题、商品、用户），暂时还没有效果。

UTM 参数（Urchin Tracking Module）

广告名称 utm_campaign

用于区分具体某个推广项目，比如某场电商、市场活动，厨房周刊某期的推广等：

utm_campaign=20160520yiguo        # 20160520 易果促销
utm_campaign=weekly236            # 厨房周刊 236 期
utm_campaign=biyeshitang-bj2016   # 毕业食堂 2016 北京站
utm_campaign=biyeshitang-sh2016   # 毕业食堂 2016 上海站
...
广告媒介 utm_medium

用于区分流量媒介，亦即流量入口具体是种什么东西，比如不同类型的广告位、微信公众号文章、push、易拉宝、合作方式等：

utm_medium=cpc                    # 为点击付费的推广链接
utm_medium=push                   # 推送
utm_medium=splash_ad              # 闪屏广告
utm_medium=launch_ad              # 开屏广告
utm_medium=issue                  # app 首页瀑布流
utm_medium=homepage_banner_ad1    # app 首页 banner #1
utm_medium=weixinmp_article       # 公众号文章
utm_medium=weixinmp_btn           # 公众号按钮
utm_medium=weibo                  # 单条微博
...

⚠️ 不同的 来源 可能有相同种类的媒介，比如 下厨房 和 一条视频 都有 公众号文章 weixinmp_article 这种媒介。

广告来源 utm_source

用于区分流量的来源，媒介由谁控制，来源就是谁，即媒介的控制者，比如下厨房、搜索引擎、合作伙伴等：

utm_source=xiachufang
utm_source=google
utm_source=baidu
utm_source=xiaomi
...

HTTPS

链接尽量使用 https:// 以防止被运营商劫持，www / m / simplecms 打头的链接都是支持 https 的。但是由于 https 的一些特性，如果页面上引用了不安全的代码或样式，页面可能会无法正常显示。在正式使用之前一定要测试下 https 链接能不能打开、显示上有没有问题。
